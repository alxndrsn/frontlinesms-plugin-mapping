Mapping

The Mapping plugin provides offline mapping functionality to geographically visualize data on a map.

What can I do with the Mapping?

Mapping offers a powerful way to visualize Forms, TextForms and SMS messages according to the sender's default location, or to synchronize with an Ushahidi deployment to allow both viewing and creation of incident reports offline. The plugin can also turn any received SMS message into an Ushahidi incident report and uploaded once an internet connection becomes available.

Note, the Mapping plugin requires some familiarity with the Ushahidi platform. You may therefore want to find out more about Ushahidi and what it's about from the Ushahidi website.

Activating the Mapping plugin

When FrontlineSMS is installed for the first time, the Mapping module is initially hidden from view. To activate the Mapping module, go to the View menu, then select Tabs, and make sure "Mapping" is ticked. A new Mapping tab should then appear with your other menus along the top of the screen. To hide the Forms module, go back into the same menu and remove the tick/check from the "Mapping" entry.

Mapping Setup

The Mapping Setup dialog provides options for the four main components: Mapping, Ushahidi, Forms and TextForms.

Mapping

  • Default Latitude - the default latitude for map center point
  • Default Longitude - the default longitude for map center point
  • Default Zoom - the default map zoom level
  • Map Provider - the map tile image provider
  • Map Cache - clear the cached map images from disk

Ushahidi

  • Source Name - the name of the Ushahidi deployment
  • Source URL - the url for the Ushahidi deployment
  • Default - the default map zoom level
  • Delete - delete the current selected deployment from the list
  • Save - save the current Source Name, Source URL and Default values
  • Cancel - disregard changes to current deployment
  • Note, upon saving a default deployment, you are prompted asking if you would to synchronize with the current Mapping setup. Clicking Yes will automatically start the synchronization process and beginning download data from the specified Ushahidi deployment.

Forms

  • Create Report Form - Generate a FrontlineForm with Ushahidi-specific fields which can be used to receive incident report details via Forms plugin.
  • Note, the button is disabled until you have successfully synchronized to an Ushahidi deployment.

TextForms

  • Create Report TextForm - Generate a TextForm with Ushahidi-specific questions which can be used to receive incident report details via TextForms plugin.
  • Note, the button is disabled until you have successfully synchronized to an Ushahidi deployment.

Mapping Synchronization

Clicking the Synchronize button will trigger the synchronization process with the current default Ushahidi deployment. The process will download the associated categories, locations and incidents for that deployment, as well as upload any incident reports that were created locally.

Note, synchronization requires an Internet connection.

Report Map

The Report Map shows the location of Message, Forms and TextForms according to their sender's default location, as well as the exact location of Ushahidi incident reports. Note, double clicking any of these map icons will show the details popup with additional information.

  • Show - the Messages, Forms, TextForms and Reports checkboxes toggle the visibility of the map icons
  • Zoom - the Zoom slider changes the map zoom level
  • Location - label showing the latitude and longitude for current mouse position
  • Categories - combobox to show reports for All Categories or just a specific category

Report List

The Report List shows a table of Ushahidi incident report information. Note, double clicking a table row will show a modal popup with that report's details.

  • Search - the search box allows fast filtering of incident reports
  • Categories - combobox to show reports for All Categories or just a specific category

Report Details

Double clicking a map icon or a table row will show a modal popup allowing you to see more information about that Report, Form or Message.

Contacts

The Mapping plugin also allows your to specify a default location for a contact, showing a Message, Form or TextForm icon on the map when one of those items is received. For example, you might want to set a hospital worker's default location to their hospital so their incoming messages can be automatically mapped.

Double clicking a contact from the list will allow you to select an Existing Location.

Or a New Location by clicking an exact latitude and longitude on the map, then providing the new location name.

Messages

The Mapping plugin allows you to turn any incoming SMS message into an Ushahidi incident report. Simply, double click a message from the list and a new incident report dialog is created pre-populated with the SMS message text, sender name and the current time.
